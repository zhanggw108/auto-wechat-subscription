这篇 Agent 论文，真正吓到我的不是它会写论文

故事是这样的。

我今天重新翻到一个项目，叫 Agent Laboratory。

它不是那种特别新的东西，论文最早在 2025 年 1 月就挂到了 arXiv 上，编号是 2501.04227，后面还进了 EMNLP 2025 Findings。

但我越看越觉得，这个东西值得重新拿出来讲。

因为它戳中的，根本不是「AI 能不能替人写论文」这个老问题。

这个问题已经被聊烂了。

真正有意思的是，它把一个学生最痛苦、最容易卡死、也最难被老师一句话讲清楚的过程，摊开了。

选题。

查文献。

设计实验。

写代码。

看结果。

再把结果写成一篇像样的论文。

你如果正在读本科、硕士，或者正在带学生，我估计你对这个流程应该一点都不陌生。

很多时候，一篇论文最难的地方不是写作。

写作反而是最后那个看得见的痛苦。

真正折磨人的，是你坐在电脑前，开着十几个 PDF，脑子里一团浆糊，不知道这个方向到底能不能做，不知道 baseline 怎么选，不知道实验跑出来的结果算不算有意义。

这种感觉太真实了。

我自己看很多学生选题的时候，也经常会遇到这种状态。

不是不努力。

是努力没有形状。

Agent Laboratory 想干的事情，就是给这种努力一个形状。

它的输入很简单，人给一个研究想法，再给一些 notes，然后系统会把这个想法推进到三个阶段。

文献综述。

实验。

报告写作。

最后输出一份研究报告和一个代码仓库。

听起来是不是很像那种，输入一句话，自动生成论文，包发顶会的民科广告？

我一开始也有点警惕。

因为这类东西太容易滑向一种很危险的叙事，仿佛科研就是把 prompt 写好，然后等 AI 给你吐一篇 paper。

但这篇论文比较有意思的地方在于，它没有把人从流程里赶出去。

恰恰相反，它一直强调，人要在每个阶段给反馈和指导。

这个细节很重要。

AI 不是在替你做研究。

它更像是在把研究过程里那些低价值、重复、耗时间、但又不能完全跳过的活，尽可能变成一条可以检查的流水线。

我说流水线不是贬义。

科研里有很多东西，本来就应该更流水线一点。

比如查一批相关论文，把每篇的核心问题、方法和局限整理出来。

比如根据一个想法，生成几套实验计划，然后让你挑哪一套更靠谱。

比如把前一次实验结果塞回上下文里，逼着下一轮实验不要装作什么都没发生。

这些事情不性感。

但很要命。

很多学生不是死在没有灵感上，而是死在这些事情没人带着做。

我一直觉得，论文训练里最稀缺的不是答案，而是过程感。

老师一句「你回去再看看文献」，学生听到的是一句话。

但真正会做研究的人，脑子里听到的是一整套动作。

去哪里搜，搜什么词，先看综述还是先看最新论文，怎么判断一篇论文是不是跟自己有关，看到一个方法之后怎么拆 baseline，怎么把疑问变成可跑的实验。

这些动作，对熟手来说像呼吸一样自然。

对新手来说，就是黑箱。

Agent Laboratory 真正有价值的地方，是它试图把这个黑箱拆开。

论文里的流程大概是这样。

文献阶段，由类似 PhD Student 的 agent 去调 arXiv API，检索论文、读摘要、拉全文、把相关文献加入综述。

实验阶段，会有机器学习工程师一类的 agent 去写代码、跑实验、解释结果。

写作阶段，再把前面的文献、计划、代码、结果和解释，组织成一份 LaTeX 报告。

中间还有 professor、postdoc、reviewer 这些角色，用来给反馈、挑问题、改方向。

你看，它其实没有发明一个神奇按钮。

它做的是，把一个研究组里本来就存在的角色和动作，压缩成一套 agent 流程。

这一下就很有意思了。

因为它让我们可以问一个更具体的问题。

不是 AI 会不会做科研。

而是，科研过程里有哪些动作，可以被拆成可验证的 agent 任务？

这个问题比前一个问题好太多了。

前一个问题太大，最后很容易变成情绪对喷。

有人说 AI 要取代科学家，有人说 AI 只会胡说八道，吵半天没啥意义。

后一个问题就实在很多。

文献检索能不能做得更稳？

实验计划能不能更完整？

代码仓库能不能跑起来？

agent 给出的实验解释，有没有引用前面的真实结果？

人类反馈插在哪个位置最有用？

这些问题都是可以做实验的。

也都是学生能拿来做毕业论文的。

说真的，这就是我对这篇论文最感兴趣的地方。

它不是给了一个完美答案。

它给了一堆可以继续追问的入口。

论文里有几个结果，挺适合公众号里保守地讲一下。

作者用不同模型驱动 Agent Laboratory，其中 o1-preview 的整体效果最好。

他们还声称，生成的机器学习代码在一些任务上能达到和已有方法相比很强的表现。

更吸引眼球的是成本，论文说相比此前的自主科研方法，Agent Laboratory 可以把研究费用降低 84%。

这个数字很漂亮。

但我建议你看这种结果的时候，先别上头。

因为科研 agent 的评估，本身就是一个很难的问题。

一篇生成出来的 paper，到底好不好？

你让自动 reviewer 打分，它可能觉得自己写得还挺不错。

你让真人研究者看，分数可能就没那么客气了。

这篇论文里其实也出现了类似现象，自动 reviewer 对生成论文贡献的判断，会比真人评价乐观很多。

这一下反而让我更信它。

因为如果一篇 agent 论文通篇都在说自己很强，完全不暴露这些尴尬，我会更慌。

科研不是 demo。

demo 里最重要的是让人觉得爽。

科研里最重要的是让人知道哪里不稳。

这也是为什么我觉得，学生读这篇论文的时候，不要把重点放在「能不能一键出论文」。

千万别这么读。

这么读就读偏了。

你真正应该看的是，它怎么把任务拆开，它怎么记录中间状态，它在哪些地方允许人介入，它的评估为什么容易被自动打分带偏。

这些东西，比那个最终生成的 PDF 重要多了。

回到学生选题这块。

如果你正在找 AI 方向的毕业论文，我会觉得 Agent Laboratory 至少能给你三个非常现实的方向。

1、做一个更窄领域的研究 agent。

不要一上来就说我要自动做科研。

这个口太大了，容易把自己噎死。

你可以选一个很小的领域，比如教育数据挖掘、医学影像分类、推荐系统里的冷启动问题，甚至只选某一类公开 benchmark。

然后让 agent 只做几件事，查文献、提 baseline、写实验计划、生成复现实验脚本。

评价也别评「论文好不好」这么虚的东西。

评更具体的。

文献召回率。

baseline 覆盖率。

实验计划可执行率。

生成代码能不能在固定环境里跑通。

这样论文就落地了。

2、做研究型 agent 的证据约束。

这块我觉得很有潜力。

因为现在很多 agent 最大的问题，不是不会说，而是说得太像真的。

尤其是科研场景，胡编一个引用、错读一个实验结果、把一个未验证的假设讲成结论，后果都很烦。

所以你可以反过来设计一个系统，不追求 agent 讲得多流畅，只追求每一句研究建议都能追到证据。

它建议你加一个 baseline，就必须说明这个 baseline 来自哪篇论文。

它建议你换一个指标，就必须说明这个指标解决什么评价盲区。

它说某个方法有效，就必须标出证据来自论文实验、作者声明，还是它自己的推测。

听着很朴素。

但科研里，朴素的东西最值钱。

3、做 human-in-the-loop 的位置实验。

这个方向也很适合学生。

因为 Agent Laboratory 里一个很核心的判断是，人类反馈能提升质量。

那问题来了。

人到底应该在哪一步介入？

一开始给想法最好？

文献综述结束后介入最好？

实验计划出来后介入最好？

结果解释时介入最好？

还是每一步都介入，反而把人累死，收益也没那么大？

这就是一个很漂亮的实验问题。

你可以把同一个研究想法分成几组流程，只改变人类反馈的位置和频率，然后比较最终实验计划、代码可运行性、报告质量和人工评分。

这个方向的好处是，它不需要你证明 AI 能完全替代研究者。

你只需要证明，在哪些节点，人和 agent 配合最划算。

我有时候觉得，AI 时代真正重要的不是自动化，而是重新分配注意力。

以前一个学生做论文，大量注意力被消耗在低层级动作里。

环境配不起来。

代码跑不通。

文献看不完。

实验记录散在不同文件夹。

结果解释每次都从头开始。

这些事情当然也重要，但它们太容易吞掉人真正应该花在问题判断上的精力。

Agent Laboratory 这类系统，如果最后真的有价值，应该不是因为它替你想出了多天才的 idea。

而是它帮你把这些杂乱动作收拢起来，让你有更多力气去问真正的问题。

这个研究问题有没有意义？

这个实验真的能证明我的观点吗？

这个结果是不是只是某个数据集上的偶然？

这个 baseline 选得公平吗？

这才是研究训练的核心。

当然，话也得说回来。

我不觉得今天的 Agent Laboratory 已经可以放心交给学生直接生产论文。

我甚至觉得，如果一个学生真的把它当成自动论文机，最后大概率会翻车。

因为它生成的东西越完整，越容易让人误以为它已经正确。

这才是最危险的地方。

一个空白文档不会骗你。

一个看起来格式齐全、有摘要、有方法、有实验、有图表的文档，会骗你。

它会让你放松警惕。

它会让你觉得，既然都这么像论文了，那应该差不多吧。

不，朋友们。

差得远。

科研里最关键的东西，往往不在格式里。

在问题有没有刺。

在实验有没有咬住问题。

在失败结果有没有被认真解释。

在你有没有勇气承认，这个方向可能暂时跑不通。

AI 可以帮你把很多东西写得像样。

但「像样」和「成立」，中间隔着一条很宽的河。

所以我对 Agent Laboratory 的态度，其实挺明确的。

它不是论文代写机。

它更像一个研究过程显微镜。

它把过去藏在导师脑子里、组会讨论里、学生草稿里的那些研究动作，尽量变成一条可以观察、可以修改、可以评价的链路。

这件事很重要。

因为当过程被摊开以后，教学才有抓手。

学生也才知道自己到底卡在哪。

是文献没查全？

是 baseline 选错了？

是实验设计太弱？

是结果解释在硬拗？

还是一开始问题就没有研究价值？

你看，这些问题都很残酷。

但残酷的问题，比温柔的空话有用。

写到这里，我突然想到一个很老的比喻。

以前学徒跟师傅学手艺，最难学的不是某个动作，而是师傅判断火候的那一下。

什么时候该停，什么时候该等，什么时候该重来。

这个东西很难写进说明书。

科研训练也一样。

真正值钱的不是某个模板，而是判断。

Agent Laboratory 这种东西，短期内不会替代判断。

但它有可能把判断发生的位置，照亮一点。

对学生来说，这就够有价值了。

如果你只是想让 AI 帮你写一篇论文，它可能会给你一个看起来很完整的幻觉。

但如果你把它当成一个研究流程拆解器，一个实验设计陪练，一个证据链检查器。

那它就突然变得有意思了。

说实话，我自己也还在摸索这类工具到底能走多远。

我不想把它吹成科研革命。

但我也不想轻飘飘地说它只是玩具。

因为很多真正改变工作流的东西，一开始看起来都挺笨的。

笨拙地拆任务。

笨拙地留记录。

笨拙地让人类一遍遍介入。

但也正是这些笨拙，让它不像一个魔法按钮，反而像一个可以被训练、被批评、被改进的研究伙伴。

这可能才是最值得期待的地方。

不是让 AI 替我们做研究。

而是让更多人，看见研究到底是怎么被做出来的。
